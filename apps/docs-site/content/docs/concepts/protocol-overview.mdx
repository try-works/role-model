# Protocol overview

The canonical protocol defines the objects a routing system must speak about consistently:

- endpoint identity
- declared capability profiles
- observed performance profiles
- capability taxonomy
- role definitions
- role bindings
- task definitions
- task execution profiles
- routing policy
- router decisions
- traces and usage

The detailed protocol pages start at [/protocol/endpoint-identity](/protocol/endpoint-identity) and continue
through [/protocol/trace-and-usage-artifacts](/protocol/trace-and-usage-artifacts).

For the protocol boundary itself, read [/introduction](/introduction).
