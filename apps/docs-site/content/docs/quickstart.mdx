# Reading path

This docs site is intentionally **not** organized as an installation walkthrough.

If you want the shortest path through the material:

1. read [What role-model defines](/introduction)
2. read [Core vocabulary](/core-vocabulary)
3. read [Protocol object model](/protocol-object-model)
4. read [How routing works end to end](/routing/how-routing-works-end-to-end)
5. use the [schema reference](/reference/canonical-schemas-reference) and
   [reason-code reference](/reference/reason-codes-and-rejection-taxonomy) while reading deeper pages

If you are debugging or reviewing routing behavior, the most useful pairing is:

- [Router decision artifact](/protocol/router-decision-artifact)
- [Eligibility and rejection](/routing/eligibility-and-rejection)
- [Decision semantics](/routing/decision-semantics)
