# Router Host Family

The router family is split into progressively heavier hosts:

- **skill router**: lightweight local or CLI-driven routing flows used for development, smoke runs, and
  protocol exercise,
- **Pi / plugin / daemon router**: future embedded or long-lived local hosts,
- **desktop router/runtime host**: future desktop-managed runtime orchestration.

The stable baseline only implements the lightweight host path. The other families are documented so the
protocol surface reserves the right extension points now.
