## Root Cause Summary

**Corrected Root Cause:** the return type for `getActiveThroughputPenaltyState()` uses an invalid indexed-access over the optional `throughputPenaltyStateByEndpointId` field, so `tsc` rejects the signature before runtime logic is even considered.

**Location:** `role-model-router/packages/core/src/router.ts:131`

**Detailed Explanation:** the function already returns either a concrete penalty-state record or `undefined`. The type annotation attempts to derive that from `RouteRequestInput["throughputPenaltyStateByEndpointId"][string] | undefined`, but TypeScript will not allow indexing into a union that still includes `undefined`. The correct fix is to use the concrete `ThroughputPenaltyStateRecord | undefined` return type instead of indexing through the optional property.

**Fix Strategy:** add a focused regression for the core build failure, run it red first, update the function signature to return `ThroughputPenaltyStateRecord | undefined`, rerun the focused core build, then rerun the full local build/lint surface before pushing again.

**Test Plan:** preserve the CI red log, add a focused Vitest regression for the core build, capture green evidence for the core build and the repo-wide local build/lint reruns, then push and re-check GitHub CI.
