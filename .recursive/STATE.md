# STATE.md

## Current State

- The repository is a pnpm workspace with shared TypeScript and Rust baselines, canonical JSON Schemas under `/protocol/schemas/`, shared root packages under `/packages/`, router packages/apps under `/role-model-router/`, fixtures under `/protocol/fixtures/` and `/testdata/`, and CI under `/.github/workflows/ci.yml`.
- Schema validation and protocol type generation are handled by `/packages/schema-tools/` and `/packages/protocol-types/`; the canonical validation path now validates `19` schema files plus `28` fixture files across valid, invalid, minimal, and edge coverage, and the committed canonical schema sources declare stable in-file `$id` values that both schema-tools and conformance fail fast on if missing or mismatched.
- The deterministic routing core lives in `/role-model-router/packages/core/` and now applies role/task/binding-aware eligibility, provider and endpoint policy filters, canonical strategy aliases, explicit exclusion codes, normalized weighted scoring, full per-candidate metric breakdowns, deterministic tie-break metadata, and stable `app_id`/`org_id` emission in router decisions.
- Fixture-driven routing conformance now lives in `/packages/conformance/src/router-fixture-conformance.test.ts` and is backed by the router golden corpus under `/protocol/fixtures/router-golden/cases/`, while `/packages/conformance/src/protocol-fixture-conformance.test.ts` and `/packages/schema-tools/` enforce the expanded schema fixture manifest.
- The observed-performance aggregation path in `/role-model-router/packages/profile-aggregator/` now uses deterministic multi-sample semantics with `measurement_window`, `endpoint_version`, benchmark/live-request source counts, failure/error-class rates, freshness/confidence formulas, and explicit runtime validation of source-count consistency.
- `/role-model-router/packages/trace/` and `/role-model-router/packages/usage/` now provide read helpers plus linkage/summarization helpers, so trace spans, trace events, usage events, and router decisions can be validated as one linked observability model.
- The smoke path in `/role-model-router/apps/gateway-smoke/` now loads a fixture-driven router case, executes the routed adapter-validation path, emits linked router-decision, request-capture, response-capture, normalized-response, adapter-diagnostics, trace, usage, and observed-performance artifacts under `/runtime-output/`, validates them against canonical schemas and linkage helpers before exit, and fails on invalid output.
- The stable config export under `/role-model-router/apps/router-devtools/` emits a normalized ACP/MCP/CLI endpoint inventory with endpoint identity fields plus declared capability metadata in `runtime-output/router-devtools/config-export.json`.
- `/role-model-router/packages/catalog/` now provides the first role-model-owned normalized catalog foundation with pinned `models.dev`-style snapshot loading, inheritance-aware normalization, provider-kind/auth-family enrichment, local override support, a vendor-version ledger derivation path, and the repo-local `catalog:export` command.
- `/testdata/catalog/` now holds the pinned catalog snapshot and local override fixtures, while `/role-model-router/packages/catalog/data/` holds the durable tracked normalized-catalog and vendor-version-ledger handoff artifacts for later router-runtime runs.
- `/role-model-router/packages/provider-account/` now provides explicit runtime-owned provider-account modeling with credential-reference parsing, auth-mode/status/health vocabularies, region/model policy fields, and validation against the tracked normalized catalog metadata.
- `/role-model-router/packages/sqlite-memory/` now provides the SQLite-first runtime-state location contract, WAL initialization, explicit runtime-memory schema, migration receipts, maintenance defaults, and provider-account persistence for same-host local-disk runtime state.
- `/role-model-router/packages/endpoint-registry/` now provides the runtime-owned endpoint registry with deterministic endpoint construction, account/model/region diagnostics, lifecycle summary output, and the repo-local `runtime:validate-registry` command.
- `/role-model-router/packages/context-envelope/` and `/role-model-router/packages/retrieval-receipt/` now provide bounded continuity-envelope assembly plus deterministic retrieval receipts over the existing SQLite runtime-state baseline.
- `/role-model-router/packages/protocol-routing/` now provides the runtime-owned projection and orchestration layer that composes endpoint-registry, context-envelope, retrieval-receipt, observed profiles, advisory routing-model inputs, and deterministic router-core scoring into one routed decision plus runtime diagnostics.
- `/role-model-router/packages/adapter-execution/` now provides the shared routed execution plane with execution-target resolution, provider-family adapter selection, capture-aware normalized execution results, trace generation, usage-event generation, and the repo-local `runtime:validate-adapter` command.
- `/role-model-router/packages/provider-openai/` and `/role-model-router/packages/provider-anthropic/` now provide the first concrete provider-family request-builder and response-normalizer implementations used by the shared adapter layer.
- `/role-model-router/apps/runtime-host-bridge/` now provides the managed bridge app that exposes `/healthz`, `/v1/models`, and `/v1/chat/completions` over the existing routed adapter-execution path and is exercised by the repo-local `runtime:validate-host` command.
- `/role-model-router/vendor/llama-swap/` is now vendored as a nested Go module and owns the local host lifecycle, request forwarding, logs, metrics, captures, and related operator surfaces while delegating routing and execution semantics to the role-model bridge.
- `/testdata/router-runtime/registry-sources.json` and `/testdata/router-runtime/context-envelope.json` now hold pinned registry/discovery and continuity-budget fixtures for the run-07 validation path.
- `/testdata/router-runtime/routing-request.json`, `/testdata/router-runtime/routing-observed-profiles.json`, `/testdata/router-runtime/routing-role-task.json`, and `/testdata/router-runtime/routing-model-guidance.json` now hold pinned run-08 routing fixtures for deterministic local/cloud runtime-routing validation.
- `/testdata/router-runtime/adapter-request.json`, `/testdata/router-runtime/adapter-routing-request.json`, `/testdata/router-runtime/adapter-role-task.json`, `/testdata/router-runtime/adapter-captures.json`, `/testdata/router-runtime/adapter-openai-response.json`, and `/testdata/router-runtime/adapter-anthropic-response.json` now hold the pinned run-09 adapter execution fixtures for deterministic routed execution validation without live provider calls.
- `/testdata/router-runtime/provider-accounts.json` now holds pinned provider-account fixtures, and the root `corepack pnpm run runtime:validate-state` command validates account configuration plus SQLite initialization/persistence against the tracked catalog handoff.
- The root `corepack pnpm run runtime:validate-routing` command now validates one end-to-end runtime-routing path and emits the router decision, retrieval-receipt summary, context-envelope summary, and routing-model diagnostics.
- The root `corepack pnpm run runtime:validate-adapter` command now validates one end-to-end routed adapter execution path and emits capture-aware execution diagnostics on top of the existing routing baseline.
- The protocol docs under `/docs/protocol/` now describe the hardened M1-M3 baseline, including `measurement_window`, role/task/binding routing semantics, trace and usage linkage, benchmark-to-profile aggregation, and the expanded reason-code vocabulary.
- `/docs/architecture/06-router-runtime-architecture-lock.md` now freezes the first router-runtime milestone as a single-host local-machine runtime built on the committed run-03 baseline, with explicit boundaries for provider taxonomy, account ownership, endpoint instantiation, protocol-driven projection, routing-model control, cache policy, OpenTelemetry interoperability, vendor/frontend/operator scope, and deferred MCP/tool work.
- `/docs/architecture/05-memory-model.md` now states that the first runtime milestone is architecturally SQLite-first and same-host/local-disk scoped even though the production runtime memory backend is still deferred.
- Future browser, edge, and native runtime families are still represented as scaffold-grade package/crate boundaries rather than production-complete implementations.
- The current validated runtime command chain is:
  - `corepack pnpm run schemas:validate`
  - `corepack pnpm run runtime:validate-routing`
  - `corepack pnpm run runtime:validate-adapter`
  - `corepack pnpm run runtime:validate-host`
  - `corepack pnpm run smoke`
- The root workspace scripts in `/package.json` now invoke nested workspace commands through `corepack pnpm ...`, which restores the canonical wrapper-path behavior used by the conformance suite and other shell-outs.
